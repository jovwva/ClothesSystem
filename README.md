# ClothesSystem

# Формат хранения данных
## ItemSO 
Хранит все данные, которые есть у предмета.
## EquippedItemSO
Хранит данные, которые нужны для визуализации предметов на пешке. Поскольку предмет может включать в себя N элементов брони, хранит в себе массив всех элементов и тип брони (броня, шлем, обувь и т.д).
## EquippedElementSO
Хранит данные, которые нужны для визуализации конкретного элемента брони. В зависимости от логики, по которой работает переодевание, его наследники хранят нужные для корректного отображения данные и тип элемента брони (левая нога, правая рука, туловище и т.д.)

# Логика работы системы

Концептуально вся логика сводится к тому, что у пешки есть “точки”, у которых мы можем менять визуал. Одевая предмет мы меняем визуальную составляющую всех “точек”, которые связаны с этой категорией предметов (например, надевая ботинки мы меняем визуал у левой и правой ноги, а надевая нагрудную броню мы меняем визуал у тела, левого и правого плеча). Поскольку существуют разные способы осуществить замену, то базовая логика реализована через абстрактные классы и обобщения.

## ElementView

Данный абстрактный класс используется для хранения ссылок на компоненты, через которые происходит обновление\замена внешнего вида элемента предмета. Потенциальный наследники должны хранить в себе ссылки на те типы данных, которые используются в их конкретной реализации. 

## ItemPresenter

Данный абстрактный класс используется для изменения предмета, который надет на персонажа. Используется как базовая абстракция, отлично подходит для предметов состоящих из одного элемента

## ItemCombinedPresenter

Данный абстрактный класс используется для изменения предмета (который состоит из N>1 элементов), который надет на персонажа. Потенциальные наследники используют конкретные реализации ElementView и EquippedElementSO для работы с различными реализациями переодевания.

# Установка

## Через Package Manager (Git URL)
- Откройте Package Manager в Unity
- Нажмите "+" → "Add package from git URL"
- Введите: https://github.com/yourusername/ClothesSystem.git

## Через manifest.json
{
  "dependencies": {
    "com.yourcompany.clothessystem": "https://github.com/yourusername/ClothesSystem.git#v1.0.0"
  }
}
